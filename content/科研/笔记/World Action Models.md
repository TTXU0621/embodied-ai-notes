# DreamZero: World Action Models are Zero-shot Policies

---

## 速览

> **一句话**：让机器人先"在脑子里放电影"（预测未来画面），再从中提取动作——而不是直接从像素跳到电机指令。

| 快速判断     | 答案                                              |
| -------- | ----------------------------------------------- |
| 解决了什么问题？ | VLA 能认东西但不会"动手"——换个环境、换个任务就废了                   |
| 怎么解决的？   | 用视频扩散模型做 backbone，同时预测画面+动作，视频充当"视觉规划器"         |
| 效果多强？    | 完全没见过的任务 39.5% vs VLA 几乎为 0；看 12 分钟人类视频就能提升 42% |
| 跟我有关吗？   | 如果你做具身智能 / 机器人学习 / 世界模型，**必读**                  |

##  为什么要做这个？（Introduction 的核心故事）

### 现有方法的问题

现有最好的机器人基础模型叫 **VLA**（Vision-Language-Action），本质是给视觉-语言大模型（VLM）接一个"动作头"来输出电机指令。

```
VLA 的思路：
  网上图片+文本 → 预训练 VLM → 加机器人数据微调 → 输出动作
```

**这个思路有个根本性缺陷**：

> VLA 的预训练数据是**静态图片**。它能学到"可乐罐长什么样""泰勒·斯威夫特是谁"，但学不到——
> - 布料被拉扯时会怎么变形？
> - 两个物体碰撞后各自怎么运动？
> - 解鞋带需要什么样的两臂协调轨迹？

论文里原话：VLA 擅长的是 **what to do**（语义层面），但缺乏 **how to do**（物理层面）。

**结果**：换个新环境、给个没练过的新任务（比如解鞋带），VLA 就完全不会了。

### 本文的洞察

> 互联网上有海量**视频**数据。视频天然编码了物理世界如何随时间演化——物体怎么掉、液体怎么流、人手怎么操作。
>
> 如果机器人模型不做"图片→动作"，而是做"视频→动作"呢？

**关键认知转变**：
- 旧范式：状态 → 动作（模仿学习，只能复制见过的行为）
- 新范式：状态 → **想象未来画面** → 从画面中提取动作（逆动力学）

> 这意味着机器人能力的上限 = 视频生成能力的上限。视频模型越好，机器人的"想象力"越强 → 泛化越好。

##  怎么做？

### 设计 1：自回归 + 用真实画面替换预测（解决"做梦做歪了"的问题）

```
训练时：        
  真实画面₁ → 真实画面₂ → 真实画面₃ → 预测画面₄ + 动作₄
  (Teacher Forcing: 前几帧用"标准答案")

推理时：
  真实画面₁ → 真实画面₂ → 真实画面₃ → 预测画面₄ + 动作₄
                                          ↓ 执行动作
                                    拿到新的真实画面₄'
                                          ↓ 替换预测，继续
  真实画面₄' → 预测画面₅ + 动作₅ → ...
```

> **聪明在哪？** 纯自回归的视频生成会"累积误差"——预测的画面越来越歪。但机器人每次执行完动作后会拿到**真实的下一帧画面**（摄像头拍的），DreamZero 用这个真实画面替换掉自己预测的，**从根本上消灭了误差累积**。这是 WAM 独有的优势——纯视频生成做不到，纯动作模型也不需要做。

### 设计 2：DreamZero-Flash（训练和推理"说实话"）

问题：视频扩散模型推理太慢（5.7 秒才能算出一个动作），机器人等不了。

直觉方案：减少去噪步数（16 步 → 1 步）。但这会导致动作质量崩塌——因为训练时模型从未见过"视频还很模糊就要输出干净动作"的情况。

**解决方案**：在训练时故意制造这种"不匹配"。

```
标准训练：
  视频噪声程度 = 动作噪声程度（始终同步）
  → 模型从未学过"视频还模糊时输出干净动作"

DreamZero-Flash：
  训练时：让视频端噪声故意更大（Beta(7,1) 分布偏向高噪声）
  动作端噪声保持均匀
  → 模型被迫学会"视频还没看清时也能猜对动作"

推理时：
  只用 1 步去噪 → 视频还不太清晰 → 但模型已经训练过这种场景 → 动作仍然准确
```

> **聪明在哪？** 这是一个典型的"让训练匹配推理"的 trick——不是发明新算法，而是发现训练和推理之间存在 mismatch，然后通过调整训练时的噪声分布来弥合。结果：推理速度 2.3×，性能几乎不降。

##  效果如何

### 结果 ①：没学过的任务，VLA 几乎全挂，DreamZero 能做

这是全文**最大的亮点**——10 个训练集里完全没有的任务：

```
"从模特头上摘帽子"   DreamZero: 85.7%   VLA: 0%
"握手"              DreamZero: 59.2%   VLA: 0%
"解鞋带"            DreamZero: 28.1%   VLA: 0%
"熨衣服"            DreamZero: 28.6%   VLA: 0%
...
10 个任务平均        DreamZero: 39.5%   VLA: 16.3%（最好基线，用了大量预训练）
                                        VLA: 0%（从零训练）
```

VLA 的本质问题是它只会复制训练数据里见过的动作模式——所以面对没见过的任务，它甚至会不管指令是什么，先伸手去抓东西再说。而 DreamZero 会'在脑子里先想象一遍'，然后在想象画面的引导下执行。"

### 结果 ②：不碰机器人，看视频就能学会

```
看 YAM 机器人做任务的视频（20 分钟，无动作标签）  → 未见任务提升 42%
看人类第一人称做任务的视频（12 分钟）              → 未见任务提升 42%
```
因为 WAM 的视频预测和动作预测是分开的——视频预测能力主要来自预训练，微调时只需要给视频数据强化'画面理解'，不需要动作标签。这意味着未来可以利用海量 YouTube 人类操作视频来训练机器人，而不用雇人一个个遥控。"

### 结果 ③：30 分钟适应一个全新的机器人身体

用 AgiBot G1 训练的模型 → 给 YAM 机器人 30 分钟"玩耍"数据 → 就能在新身体上用，还能保持零样本泛化。

##  我的思考

###  我觉得最聪明的地方

1. **"视频即规划"的哲学很优雅。** 它把机器人问题部分地"外包"给了视频生成研究——视频社区每进步一点，WAM 的泛化能力就自动提升一点。这种解耦在工程上非常聪明。

2. **用真实画面替换预测帧，消灭误差累积。** 这个设计利用了"机器人是闭环系统"这个事实，把自回归生成最大的弱点变成了优势。

###  我觉得可以质疑的地方

1. **视频预测 = 动作规划？不见得。** 论文自己承认失败主要来自视频预测出错。但"把视频画对了"和"把动作做对了"是两回事——一个好的视频预测不代表一个好的动作选择。比如视频可以预测"把杯子推倒"，但机器人不应该执行这个动作。

2. **30 分钟适应新身体的泛化边界在哪？** AgiBot G1 和 YAM 都是双臂平行夹爪——如果新身体是五指灵巧手、四足机器人、无人机呢？论文暗示"跟人越像泛化越好"，但没验证。

3. **规模化规律完全未知。** 14B > 5B，但 140B 会不会更好？数据从 500h 扩展到 50000h 呢？没有 Scaling Law，就很难判断这条路能不能走到底。

### 如果让我继续做

- **加入触觉/力觉反馈**：目前 WAM 只预测视觉和动作，但没有力信息。抓鸡蛋和抓铁球需要完全不同的力度。
- **探索 EMG 信号作为输入**：肌肉电信号比视觉更直接地反映"意图"——在人机协作或假肢控制场景中，这比纯视觉更有优势。

##  一句话带走

> **DreamZero 证明了：让机器人先"想象"未来会发生什么，再决定怎么做——比直接让它"看到就做"更聪明、更泛化。视频，就是最好的物理教科书。**
