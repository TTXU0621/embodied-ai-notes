## A. Prompt 速查清单

### S.C.A.F.F. 框架速查

| 要素 | 含义 | 一句话提醒 |
|------|------|-----------|
| **S** - Situation | 背景情境 | 你在做什么项目？用什么技术？ |
| **C** - Challenge | 具体挑战 | 现在卡在哪里？要解决什么问题？ |
| **A** - Ask | 明确请求 | 你希望 AI 做什么？输出什么？ |
| **F** - Format | 输出格式 | 要代码？要解释？要列表？ |
| **F** - Filter | 限制条件 | 什么不要做？有什么约束？ |

### 常见场景要点

| 场景 | 要点 |
|------|------|
| **创建项目** | 说清楚：做什么、给谁用、核心功能（3个以内）；先从最简单的版本开始 |
| **修改代码** | 指明位置：哪个文件、哪个函数；说清楚：现在是什么样、想改成什么样；加一句：「只改这里，其他不动」 |
| **修复报错** | 复制完整错误信息（比「报错了」有用100倍）；说明：做什么操作时出的错 |
| **解释代码** | 直接贴代码，问「这段代码在做什么」；可以追问：「为什么要这样写」 |
| **优化代码** | 说明优化目标：可读性？性能？减少重复？；让 AI 解释改了什么 |

### 提升效果的小技巧

| 技巧 | 适用场景 |
|------|----------|
| 给 1-2 个示例 | 需要更精确表达时 |
| 让 AI「想一想再回答」 | 复杂逻辑、多步骤任务 |
| 让 AI 自我检查 | 生成的代码不太放心时 |
| 分步骤提问 | 复杂任务分解更清晰 |

> **Prompt 的核心是「说清楚」，不是「套模板」。** 学会框架后，用自己的话表达就好。

## B. 常见错误速查表

### B.1 JavaScript 错误

#### undefined 错误
- **错误信息**：`Cannot read properties of undefined (reading 'xxx')`
- **通俗解释**：你在试图使用一个「不存在」的东西
- **常见原因**：变量没有赋值、数组越界、对象属性拼写错误、函数没有返回值

```
快速修复 Prompt：
"我的代码报错：Cannot read properties of undefined (reading 'xxx')
错误出现在这行：[粘贴报错的那行代码]
请帮我分析原因并修复。"
```

#### null 错误
- **错误信息**：`Cannot read properties of null (reading 'xxx')`
- **常见原因**：DOM 元素没找到（ID 错误）、API 返回 null、LocalStorage 没数据
- **修复**：添加空值判断，使用默认值如 `|| []`

#### TypeError
- **错误信息**：`xxx is not a function`
- **常见原因**：变量名覆盖了函数名、方法名拼写错误（如 `arr.pushs`）、调用了不存在的方法

#### ReferenceError
- **错误信息**：`xxx is not defined`
- **常见原因**：变量名拼写错误、作用域问题、忘记引入库

#### SyntaxError
- **常见原因**：少了括号、多了逗号、引号不匹配、JSON 格式错误（key 必须用双引号）
- **特点**：语法错误通常在代码运行前就会报错，看报错行及其上一行

#### 错误排查通用流程

```
看到红色错误 → 复制错误信息 → 看是哪一行报错
  → 能看懂？自己修
  → 看不懂？把错误信息发给 AI → AI 给出方案 → 试一试
  → 解决了？继续 / 没解决？把新错误再发给 AI
```

> **报错不可怕，复制粘贴给 AI 就行。**

### B.2 CSS/样式问题

#### 样式完全不生效 → 排查清单

| 检查项 | 怎么检查 |
|--------|----------|
| CSS 文件没引入 | 看 HTML 有没有 `<link>` 标签 |
| 路径写错了 | 检查文件路径是否正确 |
| 选择器写错了 | `.btn`（class）vs `#btn`（id），别搞混 |
| 被其他样式覆盖 | 用浏览器开发者工具检查 |
| 缓存问题 | 强制刷新（Win: `Ctrl+Shift+R` / Mac: `Cmd+Shift+R`） |

#### 布局错乱 → 快速修复

```css
/* 元素挤在一起 → 允许换行 */
.container { display: flex; flex-wrap: wrap; gap: 10px; }

/* 元素跑到屏幕外 → 限制宽度 */
.content { max-width: 100%; overflow: auto; }

/* 垂直居中不生效 → Flexbox（父容器要有高度！） */
.parent { display: flex; justify-content: center; align-items: center; height: 100vh; }
```

#### 响应式问题

1. 检查有没有加 viewport meta 标签：`<meta name="viewport" content="width=device-width, initial-scale=1.0">`
2. 有没有用百分比/rem 而不是固定 px
3. 有没有写媒体查询

```css
/* 手机优先，逐级向上 */
.container { width: 100%; padding: 15px; }

@media (min-width: 768px) {  /* 平板 */
  .container { width: 750px; margin: 0 auto; }
}

@media (min-width: 1024px) {  /* 电脑 */
  .container { width: 960px; }
}
```

#### 常见样式问题速查

| 问题 | 可能原因 | 快速解决 |
|------|----------|----------|
| 字体大小不对 | 继承了父元素样式 | 明确设置 `font-size` |
| 颜色不对 | 选择器权重不够 | 检查是否被覆盖 |
| 边距不对 | margin/padding 搞混 | margin 外边距，padding 内边距 |
| 图片变形 | 宽高比被破坏 | `object-fit: cover` |
| 按钮点不到 | 被其他元素遮挡 | 检查 `z-index` |
| 文字溢出 | 容器太小 | `overflow: hidden` 或 `text-overflow: ellipsis` |

#### 浏览器调试样式（最有效）

1. 右键页面 → 检查（或 F12）
2. 点击左上角「选择」图标，点页面元素
3. 右侧查看所有样式 —— 被划掉 = 被覆盖
4. 直接在右侧修改 CSS 实时预览
5. 调好后复制回代码文件

### B.3 交互问题

#### 按钮无反应 → 排查清单

| 检查项 | 解决方法 |
|--------|----------|
| 事件没绑定 | 检查 `onclick` 或 `addEventListener` |
| 函数名写错 | `onclick="xxx()"` 与 JS 中函数名一致 |
| JS 文件没加载 | 看控制台有没有 404 |
| JS 有语法错误 | 修复后，后面的代码才会执行 |
| 被遮挡 | 调整 `z-index` |

#### 表单提交失败

```javascript
// 阻止默认刷新
form.addEventListener('submit', function(e) {
  e.preventDefault();  // 必须有这行！
  // 然后处理数据
});

// 获取输入值
const value = document.getElementById('username').value;  // .value 不是 .innerText

// 字符串转数字
const age = Number(document.getElementById('age').value);
```

#### 数据不显示 → 排查步骤

```
数据不显示 → 数据有没有？
  → 没有 → 检查 API 返回了吗？LocalStorage 有吗？
  → 有 → DOM 更新了吗？
    → 没有 → 检查 innerHTML 设置了吗？选择器对了吗？
    → 有 → 检查 CSS：被隐藏了吗？在可视区域外？
```

#### 数据刷新后消失 → LocalStorage 完整示例

```javascript
// 保存
function saveData(data) {
  localStorage.setItem('myKey', JSON.stringify(data));
}

// 读取（带默认值）
function loadData() {
  const data = localStorage.getItem('myKey');
  return data ? JSON.parse(data) : [];
}

// 页面加载时读取
document.addEventListener('DOMContentLoaded', () => {
  const data = loadData();
  renderData(data);
});
```

#### 交互问题通用排查口诀

> **看控制台** → **加 console.log** → **检查时机** → **问 AI**

### B.4 工具问题

#### AI 不理解需求

| 原因 | 解决方法 |
|------|----------|
| 描述太模糊 | 增加具体细节 |
| 需求太复杂 | 拆分成多个小需求 |
| 缺少上下文 | 告诉 AI 已有的代码结构、技术栈 |
| 术语不准确 | 换个说法，或用示例描述 |

```
❌ "做一个好看的页面"
✅ "做一个产品展示页面。顶部大标题'智能记账本'白色字蓝色背景，
   下面3个功能介绍卡片横向排列，每个有图标+标题+描述，底部'立即下载'按钮"
```

#### AI 代码无法运行 → 排查清单

| 检查项 | 解决方法 |
|--------|----------|
| 代码不完整 | 让 AI「请给出完整代码」 |
| 缺少依赖 | 询问需要引入什么库 |
| 环境不对 | 确认是浏览器代码还是 Node.js |
| 文件结构不对 | 让 AI 说明代码放哪个文件 |

> 快速修复 Prompt：「你给的代码运行后出错了。错误信息：[粘贴]。请帮我修复，并给出完整的可运行代码。」

#### 部署失败 → 常见原因

| 错误类型 | 可能原因 | 解决方法 |
|----------|----------|----------|
| Build failed | 代码有语法错误 | 查看部署日志 |
| 404 | 文件路径问题 | 检查文件名大小写 |
| 页面空白 | JS 报错 | 打开控制台查看 |
| 样式丢失 | CSS 路径问题 | 使用相对路径 |
| API 不工作 | 环境变量没配置 | 平台配置环境变量 |

> **部署前检查清单**：所有文件已保存 ✓ / 本地测试正常 ✓ / 文件名无中文 ✓ / 路径用相对路径 ✓ / 入口是 index.html ✓

#### AI 反复给同样错误答案 → 4 个策略

1. **换个问法**：更具体的描述
2. **明确排除**：「不要使用 [之前失败的方案]」
3. **提供更多信息**：贴代码 + 报错 + 操作步骤
4. **换一个 AI 工具**：不同工具有不同知识盲区

## C. 术语表（中英对照）

### C.1 AI 编程相关

| 术语 | 一句话解释 | 类比 |
|------|-----------|------|
| **Vibe Coding** | 用自然语言告诉 AI 你想要什么，AI 生成代码 | 告诉装修师傅"我要温馨的客厅"而不是自己砌墙 |
| **Spec Coding** | 先写详细规范文档，再让 AI 按规范生成代码 | 盖房前先画好图纸 |
| **Prompt** | 你给 AI 的指令或问题 | 外卖备注："少盐少油多放辣" |
| **Context** | AI 理解你需求需要的背景信息 | 医生看病需要知道你的病史 |
| **Context Window** | AI 一次能「记住」的信息量上限 | 人的短期记忆容量 |
| **Hallucination** | AI 一本正经胡说八道 | 太想帮忙的朋友，不懂也硬编 |
| **Agent** | 能自主完成多步骤任务的 AI 系统 | 普通AI=计算器，Agent=助理 |
| **MCP** | 让 AI 连接外部工具/数据的标准协议 | USB 接口标准 |
| **Token** | AI 处理文本的基本单位 | 话费的"分钟"，用完要充值 |

> 省 Token 技巧：描述简洁但不模糊、不重复已说过信息、一次对话解决一个问题

### 图表库推荐

| 库名 | 特点 | 适合 |
|------|------|------|
| **Chart.js** | 简单易用 | 🔥 入门首选 |
| **ECharts** | 功能强大 | 复杂图表 |
| **D3.js** | 灵活度最高 | 高度定制 |
| **ApexCharts** | 现代美观 | 追求颜值 |

> **先用假数据把图表做出来，最后再接真实数据。这样迭代更快。**
